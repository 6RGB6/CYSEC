Projects

Pi-hole DNS Filtering & Network Monitoring

To reduce unnecessary telemetry and minimize ad-serving, I deployed Pi-hole on my local network using a Raspberry Pi Zero 2W. This effectively intercepts DNS queries and blocks known ad/tracking domains. It is setup as the DNS resolver for both primary and backup for every device on the network.

  • Installed and configured Pi-hole with an assigned static IP
  • Set router DHCP to use Pi-hole as primary and backup DNS
  • Routinely analyze DNS logs for anomalies
  • Created custom blocklists

Tools used: Pi-hole, Raspberry Pi OS, SSH, DNS/DHCP settings

Home Network Overhaul & Segmentation

I redesigned my home network to improve security and performance, including isolating IoT devices and monitoring traffic.

  • Architecture is now: modem ISP, router handling DHCP, DNS, SPI firewall and whitelist rules, with 6 deco mesh nodes all under 1 SSID, beamforming and fast-roaming implemented, with all DNS handled by the pi-hole.
  • This overhaul resulted in improved speeds, coverage and security without sacrificing any ease of connectivity for trusted devices by MAC address.
  • Set up VLANs to segment devices
  • Moved all IoT devices to segmented network to protect primary devices
  • Used pfSense for firewall and routing
  • Implemented traffic filtering rules and proper port policing
  • Used Wireshark and nmap to monitor activity to compare against network baseline

Tools used: pfSense, Wireshark, nmap, VLAN switch, network diagram tools
